# costscraper project

## General constraints
- only read and write in the project directory (costscraper) and its subdirectories.

## Technology and tooling
- Python 3 project using `uv` for package management.
- No `pyproject.toml`, `src/`, or `tests/` exist yet; create them under the root when adding code.

## Project structure
- Place all source code in `src/`.
- Place all tests in `tests/`.

## Coding conventions
- Use **camelCase** for functions, classes, and variable names. (Do not default to Python snake_case.)
- Use named parameters in function definitions and calls where practical.

## Quality constraints
- Maintain test coverage of at least 70%.

## License constraints
- The project is under an Apache Commons License.
- Do not add dependencies incompatible with the Apache Commons License.
- Avoid dependencies with a GPL license.
